## 一、问题定义与背景

### 1.1 问题全称

- **英文**: Two-Dimensional Cutting Stock Problem (2D-CSP)
- **中文**: 二维下料问题 / 二维切割库存问题
- **别名**: 2D Bin Packing Problem (2D-BPP) 的变体

### 1.2 工业背景

二维下料问题广泛存在于制造业中:

| 行业 | 母板材料 | 子板产品 |
|------|----------|----------|
| 钢铁 | 钢板 | 零部件毛坯 |
| 玻璃 | 平板玻璃 | 窗户、镜子 |
| 木材 | 胶合板 | 家具部件 |
| 纺织 | 布卷 | 服装裁片 |
| 造纸 | 纸卷 | 包装纸箱 |

### 1.3 问题描述

**输入**:
- 母板 (Stock): 固定尺寸 W x L (宽度 x 长度)
- 子板 (Items): N种规格，每种 i 有:
  - 尺寸: w_i x l_i
  - 需求量: d_i

**输出**:
- 切割方案: 如何切割母板以满足所有子板需求

**目标**:
- 最小化使用的母板数量

### 1.4 切割约束: 两阶段直通切 (Two-Stage Guillotine Cutting)

本项目采用**两阶段直通切**模型，这是工业中最常见的切割方式:

```
Stage 1 (第一阶段 - 宽度方向):
┌─────────────────────────────┐
│                             │
│   母板 (W x L)              │
│                             │
└─────────────────────────────┘
          ↓ 垂直切割
┌───────┬───────┬─────────────┐
│条带1  │条带2  │  条带3      │
│(w1×L) │(w2×L) │  (w3×L)     │
└───────┴───────┴─────────────┘

Stage 2 (第二阶段 - 长度方向):
对每个条带进行水平切割
┌───────┐
│ 子板A │
├───────┤
│ 子板B │
├───────┤
│ 子板C │
├───────┤
│ 废料  │
└───────┘
```

**关键约束**:
1. **直通切 (Guillotine Cut)**: 每次切割必须从一边贯穿到另一边
2. **两阶段**: 先垂直后水平，或先水平后垂直
3. **不允许旋转**: 子板方向固定，不能旋转90度

## 二、数学模型

### 2.1 集合与参数

**集合**:
- I = {1, 2, ..., N}: 子板类型集合
- J = {j : j是某个子板宽度}: 条带类型集合 (|J| = 唯一宽度数)
- K: 母板切割模式集合 (动态生成)
- P: 条带切割模式集合 (动态生成)

**参数**:
| 符号 | 含义 | 单位 |
|------|------|------|
| W | 母板宽度 | 长度单位 |
| L | 母板长度 | 长度单位 |
| w_i | 子板i的宽度 | 长度单位 |
| l_i | 子板i的长度 | 长度单位 |
| d_i | 子板i的需求量 | 整数 |
| w_j | 条带j的宽度 | 长度单位 |

### 2.2 决策变量

| 变量 | 类型 | 含义 |
|------|------|------|
| Y_k | 非负整数 | 母板模式k的使用次数 |
| X_p | 非负整数 | 条带模式p的使用次数 |

### 2.3 模式系数

| 系数 | 含义 |
|------|------|
| C_jk | 母板模式k产出j型条带的数量 |
| B_ip | 条带模式p产出i型子板的数量 |
| A_jp | 指示模式p属于哪种条带类型 (0或1) |

### 2.4 主问题 (Master Problem)

```
目标函数:
    min  Σ_k Y_k                                    (1)

约束条件:
    Σ_k C_jk·Y_k - Σ_p A_jp·X_p ≥ 0    ∀j∈J       (2) 条带平衡
    Σ_p B_ip·X_p ≥ d_i                  ∀i∈I       (3) 需求满足
    Y_k ≥ 0, integer                    ∀k∈K       (4)
    X_p ≥ 0, integer                    ∀p∈P       (5)
```

**约束解释**:
- (1) 最小化母板使用总数
- (2) 产出的条带数 ≥ 消耗的条带数 (条带可以剩余)
- (3) 产出的子板数 ≥ 需求量

### 2.5 LP松弛

将整数约束 (4)(5) 松弛为连续变量:

```
Y_k ≥ 0    (无整数约束)
X_p ≥ 0    (无整数约束)
```

LP松弛的最优值提供**下界** (Lower Bound)。

## 三、Branch-and-Price算法原理

### 3.1 算法框架

Branch-and-Price = **列生成 (Column Generation)** + **分支定界 (Branch-and-Bound)**

```
算法流程:
┌─────────────────────────────────────────────────┐
│ 1. 初始化                                        │
│    - 生成初始可行解 (启发式)                      │
│    - 初始化分支树 (根节点)                        │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│ 2. 根节点列生成                                  │
│    循环:                                         │
│    ├─ 求解受限主问题 (RMP) → 得到对偶值          │
│    ├─ 求解定价子问题 (SP1, SP2)                  │
│    ├─ 如果有改进列 → 加入RMP，继续循环           │
│    └─ 否则 → 列生成收敛，得到LP下界              │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│ 3. 整数性检查                                    │
│    - LP解是整数? → 找到最优解                    │
│    - LP解有分数? → 需要分支                      │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│ 4. 分支定界                                      │
│    循环:                                         │
│    ├─ 选择分数变量/弧进行分支                    │
│    ├─ 创建子节点                                 │
│    ├─ 对每个子节点进行列生成                     │
│    ├─ 更新上界 (如果找到整数解)                  │
│    ├─ 剪枝 (下界 ≥ 当前最优上界)                 │
│    └─ 直到所有节点处理完毕                       │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│ 5. 输出最优解                                    │
└─────────────────────────────────────────────────┘
```

### 3.2 列生成原理

**核心思想**:
模式空间 K 和 P 是指数级的，无法预先枚举。列生成动态生成有用的模式。

**Dantzig-Wolfe分解**:
1. 求解**受限主问题 (RMP)**: 只包含部分模式的LP
2. 获取**对偶值**: 约束(2)的对偶值 v_j, 约束(3)的对偶值 π_i
3. 求解**定价子问题**: 寻找**减少成本 (Reduced Cost)** 为负的新模式
4. 如果找到 → 加入RMP，重复
5. 如果没有 → 当前解是LP最优

### 3.3 对偶变量含义

| 对偶变量 | 对应约束 | 经济含义 |
|----------|----------|----------|
| v_j | 条带平衡 (2) | j型条带的"影子价格" |
| π_i | 需求满足 (3) | i型子板的"边际价值" |

### 3.4 减少成本计算

**Y-列 (母板模式k) 的减少成本**:
```
rc_Y = 1 - Σ_j v_j · C_jk
```
- 系数1来自目标函数 (每使用一块母板增加1)
- 如果 rc_Y < 0，模式k能改进目标

**X-列 (条带模式p, 属于条带类型j) 的减少成本**:
```
rc_X = -v_j + Σ_i π_i · B_ip
     = Σ_i π_i · B_ip - v_j
```
- 如果 rc_X > v_j，即 profit > v_j，模式p能改进目标

## 四、定价子问题详解

### 4.1 SP1: 宽度方向子问题 (母板 → 条带)

**问题**: 如何切割母板宽度W，产出各类型条带，使总价值最大

**模型**:
```
max  Σ_j v_j · G_j              (条带总价值)
s.t. Σ_j w_j · G_j ≤ W          (宽度容量约束)
     G_j ∈ Z⁺                    (整数个条带)
```

**本质**: 一维整数背包问题 (1D Knapsack)
- 容量: W (母板宽度)
- 物品: 各类型条带j
- 物品尺寸: w_j (条带宽度)
- 物品价值: v_j (对偶值)

**收敛条件**:
当最优解的目标值 ≤ 1 时，没有改进的Y-列

### 4.2 SP2: 长度方向子问题 (条带 → 子板)

**问题**: 对于每种条带类型j，如何沿长度L切割，产出子板，使总价值最大

**模型** (对每个条带类型j):
```
max  Σ_{i∈I(j)} π_i · D_i - v_j    (子板价值 - 条带成本)
s.t. Σ_{i∈I(j)} l_i · D_i ≤ L      (长度容量约束)
     D_i ∈ Z⁺                       (整数个子板)
```

其中 I(j) = {i : w_i ≤ w_j} 表示能放入j型条带的子板集合

**本质**: 一维整数背包问题
- 容量: L (条带/母板长度)
- 物品: 能放入该条带的子板i ∈ I(j)
- 物品尺寸: l_i (子板长度)
- 物品价值: π_i (对偶值)

**收敛条件**:
当所有SP2的最优利润 ≤ v_j 时，没有改进的X-列

### 4.3 子问题求解方法对比

| 方法 | 英文名 | 时间复杂度 | 优点 | 缺点 | 适用场景 |
|------|--------|------------|------|------|----------|
| CPLEX IP | Integer Programming | O(2^n) worst | 通用，保证最优 | 慢，开销大 | 小规模或调试 |
| Arc Flow | 弧流模型 | O(n·C) | 支持弧约束，快 | 实现复杂 | 需要弧分支时 |
| DP | 动态规划 | O(n·C) | 极快，内存效率 | 不支持弧约束 | 根节点CG |

### 4.4 弧流网络模型 (Arc Flow Model)

**目的**: 将背包问题建模为网络流，支持弧分支

**SP1弧流网络**:
```
节点: 0, 1, 2, ..., W (位置节点)
弧:
  - 物品弧: (t, t+w_j) 表示在位置t放置j型条带
  - 损耗弧: (t, t+1) 表示损失1单位宽度

从节点0到节点W的路径 = 一个切割模式
```

**示例** (W=10, 条带宽度 w1=4, w2=3):
```
0 ──[w1=4]──> 4 ──[w2=3]──> 7 ──[loss]──> 8 ──[loss]──> 9 ──[loss]──> 10
模式: 1个w1条带 + 1个w2条带 + 3单位损耗
```

## 五、求解难度因素分析

### 5.1 难度因素总览

| 排名 | 因素 | 权重 | 影响机制 | 可控程度 |
|------|------|------|----------|----------|
| 1 | 子板-母板尺寸比 | 35% | 装填灵活性 | 高 |
| 2 | 子板种类数 | 25% | 组合复杂度 | 高 |
| 3 | 需求量分布 | 20% | 整数近似 | 中 |
| 4 | 尺寸异质性 | 15% | 模式多样性 | 中 |
| 5 | 宽度种类数 | 5% | SP2子问题数 | 低 |

### 5.2 因素1: 子板-母板尺寸比 (Size Ratio) - 最关键

**定义**:
```
面积比: size_ratio = avg(w_i × l_i) / (W × L)
宽度比: width_ratio = avg(w_i) / W
长度比: length_ratio = avg(l_i) / L
```

**影响机制**:
- **小子板** (ratio < 10%): 每块母板可装填20+个子板 → 灵活组合 → 易找好解
- **大子板** (ratio > 30%): 每块母板仅装2-4个子板 → 选择受限 → LP松弛差

**难度梯度表**:

| 尺寸比 | 每板装填数 | LP-IP Gap | 典型节点数 | 难度等级 |
|--------|------------|-----------|------------|----------|
| < 5% | > 20 | < 1% | < 10 | 极易 |
| 5-10% | 10-20 | 1-2% | 10-50 | 简单 |
| 10-20% | 5-10 | 2-5% | 50-200 | 中等 |
| 20-35% | 3-5 | 5-15% | 200-1000 | 困难 |
| 35-50% | 2-3 | 15-30% | 1000-5000 | 很难 |
| > 50% | 1-2 | 30%+ | 5000+ | 极难 |

**数学解释**:
```
LP松弛允许使用分数块母板:
  - 如果最优需要3.2块母板，LP解=3.2，整数解≥4
  - Gap = (4-3.2)/4 = 20%

子板越大 → 每块母板装填越少 → 分数部分影响越大 → Gap越大
```

### 5.3 因素2: 子板种类数 (Number of Item Types)

**影响机制**:
- 更多种类 → 更大的模式空间 → 列生成收敛慢
- 更多种类 → 更多SP2子问题实例 → 每轮定价更耗时
- 更多种类 → 更复杂的整数组合 → 分支树更大

**难度梯度表**:

| 种类数N | 模式空间估计 | CG迭代数 | 难度等级 |
|---------|--------------|----------|----------|
| 5-10 | O(10^3) | 5-10 | 简单 |
| 10-20 | O(10^5) | 10-20 | 中等 |
| 20-40 | O(10^8) | 20-40 | 困难 |
| 40-60 | O(10^12) | 40-80 | 很难 |
| > 60 | O(10^15+) | 80+ | 极难 |

### 5.4 因素3: 需求量分布

**关键指标**:
- **总需求量**: Σd_i - 决定解的规模
- **平均需求**: avg(d_i) - 影响模式复用
- **需求方差**: var(d_i) - 影响分布特性

**影响分析**:

| 需求特征 | 描述 | 难度影响 | 原因 |
|----------|------|----------|------|
| 高需求 | d_i > 10 | 降低难度 | 同一模式多次复用 |
| 低需求 | d_i = 1-3 | 增加难度 | 几乎每种模式用一次 |
| 均匀分布 | 所有d_i相近 | 中等 | 平衡 |
| 偏斜分布 | 少数d_i很大 | 可能降低 | 主导项优先优化 |

**数学解释**:
```
高需求时:
  - 最优模式可以重复使用多次
  - 减少了需要生成的不同模式数
  - LP-IP Gap中整数舍入的影响被稀释

低需求时:
  - 每种子板需求1-2个
  - 几乎无法复用模式
  - 整数舍入误差累积
```

### 5.5 因素4: 尺寸异质性 (Size Heterogeneity)

**定义**: 子板尺寸的分散程度

**计算方法** (变异系数):
```
CV_width = std(w_i) / mean(w_i)
CV_length = std(l_i) / mean(l_i)
CV_combined = sqrt((CV_width)^2 + (CV_length)^2) / sqrt(2)
```

**影响机制**:

| CV值 | 尺寸特征 | 难度影响 | 原因 |
|------|----------|----------|------|
| < 0.15 | 高度相似 | 较易 | 尺寸相近，易紧密组合 |
| 0.15-0.30 | 中等差异 | 中等 | 平衡 |
| 0.30-0.50 | 较大差异 | 较难 | 难以找到互补组合 |
| > 0.50 | 极端差异 | 很难 | 尺寸跨度大，浪费严重 |

### 5.6 因素5: 质数偏移效应 (Prime Offset Effect)

**现象**: 当子板尺寸与母板尺寸不成整除关系时，产生不可避免的浪费

**示例分析**:
```
母板宽度 W = 200

情况A: w = 50 (整除)
  200 / 50 = 4 (正好)
  每行放4个，利用率100%

情况B: w = 67 (接近质数)
  200 / 67 = 2.98...
  最多放2个，占用134，剩余66
  利用率 = 134/200 = 67%

情况C: w = 63 (3的倍数)
  200 / 63 = 3.17...
  最多放3个，占用189，剩余11
  利用率 = 189/200 = 94.5%
```

**难度影响**:
- 质数偏移导致**固有浪费**
- 浪费空间无法被其他子板利用
- LP松弛无法体现整数约束的刚性 → Gap增大

### 5.7 宽度种类数的隐藏影响

**定义**: J = {unique widths} 中的元素个数

**影响**:
```
|J| 大:
  - 更多种类的条带
  - 更多SP2子问题实例
  - 条带平衡约束更多

|J| 小:
  - 条带类型少
  - 多种子板共用条带类型
  - 简化了问题结构
```

**计算**:
```
宽度多样性 = |J| / N = 唯一宽度数 / 子板种类数
```

## 六、难度评分公式与Gap预估

### 6.1 综合难度评分公式

基于上述分析，建议使用以下加权公式:

```
DIFFICULTY_SCORE =
    0.35 × f_size(size_ratio)
  + 0.25 × f_types(N)
  + 0.20 × f_demand(avg_demand)
  + 0.15 × f_cv(CV_combined)
  + 0.05 × f_width_div(width_diversity)
```

**各因子函数定义**:

```
f_size(r) = r / 0.20                           # 以20%为基准
f_types(N) = N / 30                            # 以30种为基准
f_demand(d) = 5.0 / d                          # 需求越低越难
f_cv(cv) = cv / 0.30                           # 以0.30为基准
f_width_div(wd) = wd                           # 直接使用
```

### 6.2 难度等级划分

| 得分范围 | 难度等级 | 预估Gap | 预估节点数 | 60s内求解 |
|----------|----------|---------|------------|-----------|
| < 0.5 | 极易 (Trivial) | < 1% | < 10 | 必定完成 |
| 0.5-0.8 | 简单 (Easy) | 1-3% | 10-100 | 很可能完成 |
| 0.8-1.2 | 中等 (Medium) | 3-8% | 100-500 | 可能完成 |
| 1.2-1.6 | 困难 (Hard) | 8-15% | 500-2000 | 不一定完成 |
| 1.6-2.0 | 很难 (Very Hard) | 15-25% | 2000-10000 | 可能超时 |
| > 2.0 | 极难 (Expert) | > 25% | > 10000 | 很可能超时 |

### 6.3 Gap预估公式

基于难度得分直接预估Gap:

```
estimated_gap =
    if score < 0.5:  1%
    elif score < 0.8:  1% + (score-0.5) × 6.67%
    elif score < 1.2:  3% + (score-0.8) × 12.5%
    elif score < 1.6:  8% + (score-1.2) × 17.5%
    elif score < 2.0:  15% + (score-1.6) × 25%
    else:  25% + (score-2.0) × 20%
```

### 6.4 预估精度说明

上述公式基于算法分析和经验判断，实际Gap会受到:
1. 具体实例的随机性
2. 求解器参数设置
3. 时间限制
4. 初始解质量

**建议**: 生成算例后进行实际求解校准，调整公式参数

## 七、算例生成参数配置建议

### 7.1 预设档位配置

**Easy (简单)**:
```
num_types = 8
stock = 200 × 400
size_ratio = 8-12%
avg_demand = 12-18
```

## 八、实验验证与校准

### 8.1 校准数据收集

**需要记录的求解结果**:
```cpp
struct SolveResult {
    // 输入特征
    int num_types;
    double avg_size_ratio;
    double avg_demand;
    double size_cv;
    double width_diversity;

    // 求解结果
    double root_lb;        // LP下界
    int best_int;          // 最优整数解
    double gap;            // (best_int - root_lb) / best_int
    int node_count;        // 分支节点数
    double solve_time;     // 求解时间(秒)
    bool time_limit_hit;   // 是否超时
};
```

### 8.2 校准流程

```
1. 生成覆盖各难度等级的算例 (每级10-20个)
2. 运行求解器，记录结果
3. 拟合公式参数:
   - 调整权重 w_size, w_types, w_demand, w_cv, w_width
   - 调整Gap预估曲线
4. 验证准确性:
   - 计算预估Gap与实际Gap的RMSE
   - 目标: RMSE < 5%
```

### 8.3 预期校准结果

校准后的公式应满足:
- 80%的算例: |预估Gap - 实际Gap| < 5%
- 95%的算例: |预估Gap - 实际Gap| < 10%
- 难度等级预测准确率 > 85%
